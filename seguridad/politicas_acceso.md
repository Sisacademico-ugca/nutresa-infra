#Politicas de Acceso

## Firewall
-Todo trafico entrante es bloqueado por defecto
-Solo se permiten puertos:
- 22 SSH
- 80 HTTP
- 443 HTTPS
- 2049 NFS

## Usuarios 
- Cada administrados tiene usuario individual.
- No se permite uso compartido de credenciales.
- Contraseñas seguras obligatorias.

## Permisos 
- Uso de SETUID para tareas especificas.
- Uso de SETUID en carpetas colaborativas.
- Uso de Sticky Bit para proteger archivos compartidos.

## SSH
- Acceso remoto restringido.
- Solo usuarios autorizados pueden conectarse.

## Backups
-Respaldos automaticos diarios.
